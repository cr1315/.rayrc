01.
環境汚染からの解放

仕事やプライベートで扱うLinux VMは無数にある。 セキュリティポリシーが厳しい環境では、新しいツールのインストールさえも「環境汚染」と見なされかねない。

だからこそ、我々は「自分の領地」を持つ必要がある。

.rayrc は、すべてをホームディレクトリ配下、あるいは自分だけの管理領域に閉じ込める。 他者の環境を壊さず、しかし自分の生産性は極限まで高める。 それが、このツールの哲学だ。

# Directory Structure
~/.rayrc/
├── install // Idempotent script
├── bash/
│ ├── aliases.sh
│ └── functions.sh
├── vim/
│ └── .vimrc
└── bin/ // Local binaries

Why .rayrc?

単なるdotfilesではない。これはInfrastructure as Codeの実践だ。

Self-Contained

システム領域（/usr/bin等）を汚さない。全ての設定とツールは自分のディレクトリ内で完結。 管理者権限がなくても、最強の環境を構築可能。

Idempotent

何度実行しても壊れないインストールスクリプト。 状態をチェックし、必要な差分だけを適用するIaCの思想をシェルスクリプトで実現。

Pluggable

必要なのはBashだけ？それともVimも？ モジュール構造により、必要な機能だけをロード。重厚長大な設定ファイルとはおさらば。
